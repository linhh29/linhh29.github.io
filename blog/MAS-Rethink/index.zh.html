重构自动化多智能体系统

从幻象到进化：构建更优多智能体系统的统一视角

本文将我们关于自动化多智能体系统（MAS）的三项相互关联的研究串联为一个统一视角，系统诊断了当前自动化 MAS 为何表现不佳，并围绕领域深度与进化广度提出原则性的解决方案，为基于大语言模型的多智能体协作指明未来的设计方向。

序章：MAS 的承诺

大语言模型（LLMs）的快速演进从根本上改变了人工智能的格局。**基于 LLM 的多智能体系统（MAS）** 作为一种强大的范式应运而生，旨在通过协作智能处理复杂任务——包括任务分解、角色专业化、并行执行、辩论及交叉验证。通过编排多个具备不同角色与能力的 LLM 智能体，MAS 得以应对超出任何单一模型能力范围的问题。

传统上，设计有效的 MAS 需要人类专家投入大量手工工程。然而，近年来研究社区已转而拥抱自动化 MAS 生成范式——即通过框架自主发现智能体角色、通信拓扑与执行工作流，无需人类干预。其愿景是：让强大的 LLM 分析任务与轨迹，挖掘出超越人工设计方案的创新智能体工作流。

但这一愿景是否经得起推敲？过去数月中，我们系统地审视了自动化 MAS——从严格诊断其失败原因，到提出原则性的解决方案。本文将三项研究整合为一个统一视角：自动化 MAS 为何失败、如何赋予其领域专长（领域深度），以及如何让其持续进化（进化广度）。

第一部分：多智能体优势的幻象

核心问题：自动化 MAS 真的有效吗？

主流观点认为多智能体系统优于单智能体系统（SAS），理由包括上下文保护、并行处理和分布式决策等优势。 这一预期推动了自动化 MAS的快速发展，涌现出 DyLAN、MAS-Zero、ADAS、AFlow、MaAS 和 MAS-Orchestra 等框架， 它们能够动态分解任务、配置智能体角色并管理执行流程，全程无需人类干预。

我们针对这六个代表性的自动化 MAS，与一个简单的 SAS 基线——带自一致性的思维链（CoT-SC）——展开了严格的系统性评估。 评估覆盖多个 LLM，包括 GPT-4o、GPT-5、GPT-OSS-120B 和 Gemini-2.5-Pro， 并涵盖标准推理任务（GPQA-Diamond、HLE-Maths、SWE-Bench Lite）和复杂的智能体设置（BrowseComp-Plus）。

结果：CoT-SC 在大多数情况下优于自动化 MAS

结果令人警醒。在几乎所有基准测试中，CoT-SC 通常优于自动化 MAS 框架， 往往在不到 10% 的计算成本下达到更高的准确率。该评估得出三项关键发现：

  • 1. 中端模型无增益：自动化 MAS 未能为 GPT-4o 或 GPT-OSS 等模型带来一致的改进。
  • 2. 模型层级优势：使用 CoT-SC 的单智能体 GPT-5 实例始终优于最复杂的基于 GPT-4o 的自动化 MAS，同时消耗不到一半的总 token 数。
  • 3. 存在能力门槛：显著的自动化 MAS 提升仅出现在使用 GPT-5 和 Gemini-2.5-Pro 的 HLE-Math 上，表明架构复杂性需要一定的基线推理能力才能体现优势。

SMFR 诊断：任务适用性并非问题所在

一个常见的质疑是，标准基准缺乏 MAS 展现优势所需的结构属性。为回应这一质疑，我们引入了 合成多跳金融推理（SMFR）数据集——一个诊断性基准，明确具备子任务分解、 上下文密集型输入以及清晰的并行化与专业化机会，与 Anthropic 关于 MAS 适用场景的指导原则一致。 每个问题都需要多步上下文提取、数值推理和跨投资者综合，且在设计中刻意排除了捷径策略。

即使在明确的智能体需求下，自动化 MAS 也始终无法超越 CoT-SC 的效率。在 SMFR 上，GPT-5 搭配 CoT-SC 仅达到 49.1% 的准确率， 而我们的 Expert-MAS——借助 Python 执行器强制实施显式分解、角色专业化与确定性编排——达到了 96.7%。 这说明 MAS 范式本身是可行的，但当前的自动化 MAS 始终无法把握任务特定的结构性机会。

架构解构：内部出了什么问题？

为探究自动化 MAS为何表现不佳，我们系统性地解构了这六个自动化 MAS 生成的架构。 三种系统性的失败模式浮出水面，各自折射出架构复杂度与功能效用之间的根本性错位：

功能崩溃与角色冗余

在 DyLAN 中，超过 90% 的 GPT-5 案例中，智能体几乎立刻达成共识，退化为冗余的 CoT-SC 基线。在 MAS-Zero 中，验证器存在系统性位置偏差——超过 45% 的情况下优先选择上下文窗口中靠前的条目，导致后续搜索轮次沦为对最终输出几乎无因果影响的"昂贵旁观者"。

收敛于启发式伪迹

ADAS 和 AFlow 实质上沦为启发式探索者，而非原则性优化器。ADAS 展现出非单调的搜索动态，准确率往往在早期触及峰值后随即退化。AFlow 的"优化"工作流退化为平凡的集成模式：在五个数据集上评估的 14 个最终工作流中，50% 仅是将单个自定义提示迭代三次后做聚合，结构上与 CoT-SC 毫无二致。

动态路由中的激励错位

MaAS 在上下文密集型任务上退化为平凡的 I/O 调用（BrowseComp-Plus 上 74.2% 的激活），背后是成本主导优化的驱动。MAS-Orchestra 完全忽略其多样化的智能体池，无论任务难度如何，都收敛于 Debate 和 Reflexion 两类智能体的僵硬二元偏好，背离了其智能路由的设计初衷。

集成陷阱：这种崩溃的主要驱动因素是对 CoT 和 CoT-SC 的依赖——它们被当作自动化 MAS 的基本构建块。 这些框架未能实现真正的协同，反而收敛于结构退化——在"优化图"的表象下，只是为了不断重新发现基本的集成模式。 约 10 倍的计算成本增量，换来的不过是一个冗余、路由混乱的标准 CoT-SC 变体。

第二部分：Unified-MAS：掌握领域深度

知识缺口：为何自动化 MAS 在专业领域失败

第一部分表明，自动化 MAS 存在严重的架构退化问题。当应用于知识密集型领域（如医疗、 法律和金融）时，现有框架面临根本性瓶颈。它们要么依赖缺乏专业知识的通用节点静态库（如思维链、辩论）， 要么尝试动态生成节点——但编排器受内部参数化知识的限制，不得不同时设计领域专用逻辑并优化高层拓扑。 这种架构耦合严重损害了系统的实际效能。

核心洞见：将"做什么"与"如何做"解耦

我们提出Unified-MAS，通过离线节点合成将细粒度节点实现与拓扑编排解耦。 与让编排器同时承担设计领域专用智能体和优化其连接关系的任务不同，Unified-MAS 充当 离线节点合成器，在任何拓扑搜索开始之前生成专家级节点。该框架分两个阶段运行：

阶段一 · 关键词提取

采样验证实例，沿七个维度提取关键词：领域、任务、实体、动作、约束、期望结果和隐性知识。

阶段一 · 策略搜索

四种搜索策略分别针对背景知识（Scholar）、系统架构（Google）、代码仓库（GitHub）和评估基准，将节点合成扎根于真实领域知识。

阶段二 · 奖励评分

在验证数据上执行节点，计算困惑度引导的奖励，衡量每个节点对最终答案的改进和一致性贡献。

阶段二 · 瓶颈优化

每轮识别最低奖励节点，优化其提示或添加子调用，重复直至性能收敛，通常在 10 轮内完成。

结果：更优的成本-性能权衡

Unified-MAS 扮演了通用节点合成器的角色。生成的节点可直接插入 MAS-Zero、AFlow、 ScoreFlow 和 MAS²，无需修改其高层编排器，将通用自动化 MAS 提升为领域专家—— 在四个 LLM 骨干上均取得了稳定收益。以 J1Bench（法律判决）为例， Unified-MAS 生成了如下结构化的司法流水线： 法律要件提取器 事实分析器 责任推理器 损害赔偿计算器 判决起草器，取代了动态生成方法通常产出的表面集成节点（"Expert1"、"Expert2"）：

关键发现

外部检索是关键的区分因素：它将节点生成从幻觉命名转化为根植于真实领域知识的专家工作流设计。生成的节点蕴含了可追溯、立足于领域的推理阶段。

核心洞见

Unified-MAS 不局限于任何单一设计 LLM。Gemini 模型合成简洁的宏观级工作流（约 5 至 6 个节点），而 GPT-5-Mini 偏好微观级粒度（约 10 个节点）：Unified-MAS 在所有骨干上均持续驱动显著收益。

第三部分：Skill-MAS：掌握进化广度

能力与经验的两难困境

第一部分验证了 MAS 范式本身是可行的，但当前的自动化 MAS 始终无法发现能真正发挥多智能体结构优势的架构。 自动化 MAS 面临着能力与经验的两难困境：

推理时自动化 MAS

EvoAgent、AOrchestra 和 AFlow 利用冻结前沿 LLM 的强大推理能力，但元智能体每轮执行相同的搜索例程，没有累积记忆。它无法从过去的失败中学习，也无法跨运行传递诊断经验。

训练时自动化 MAS

MAS² 和 MAS-Orchestra 通过微调小型 LLM（通常为 7B 参数）将编排能力内部化，但受限于较低的能力上限，且扩展到前沿模型（>100B）的代价高昂——而最强的推理能力恰恰集中于这些模型中。

是否存在第三条路径？一条将经验保留与参数更新解耦的路径， 让冻结的前沿元智能体在多样化任务中逐步积累经验，持续优化其编排能力？

第三条路径：元技能作为可进化的产物

我们提出Skill-MAS，将元智能体的高层编排能力抽象为 可进化的元技能——一个结构化的自然语言产物（SKILL.md），蕴含了跨越整个编排流水线的 可复用、策略级原则。元技能组织为三个模块，把握的是架构性知识，而非对任务特定解法的过拟合：

任务分解（做什么） 智能体工程（谁来做） 工作流编排（如何做）

阶段一 · 多轨迹推演

在当前元技能下对每个验证任务采样 K 条独立轨迹，计算每个任务的不确定性（分数标准差）和难度（负均值分数）以刻画行为分布。

阶段二 · 优先级选择

将不确定性和难度融合为统一的优先级分数，排序任务，应用肘部检测选择最具信息量的子集进行反思，集中优化预算。

阶段二 · 层次化反思

任务内对比分析比较高分与低分轨迹；跨任务综合提取重复出现的失败模式，将有影响力的修补方案排序为结构化证据。

阶段二 · 技能优化

优化器在保留三模块框架的前提下重写元技能，移除无效规则，添加可泛化的编排原则，并通过结构性有效性检查。

结果：更优的成本-性能与可迁移技能

即便不做迭代进化，Skill-MAS 初始版已展现出有竞争力的性能——例如，使用 Gemini-3.1-Flash 时， 它略微超越了最佳基线 AFlow（平均 21.68 vs. 21.29）。Skill-MAS 优化版在几乎所有设置中 均以显著优势超越全部基线，在四个 LLM 上均达到了最高平均性能。使用 DeepSeek-V4-Flash， 平均性能从 33.72% 提升至 41.05%；Qwen3.5-Plus 则从 32.61% 提升至 38.41%。

成本-性能权衡展现出三种范式之间的清晰分野：训练时 MAS 成本最低但性能最差； 推理时 MAS 以最高的推理成本换取精度提升。Skill-MAS 占据了更优的位置——通过进化后的元技能 一次性生成 MAS，而非对每个样本重新迭代优化，以中等成本取得了最佳平均性能。

为评估泛化能力，我们分析了元技能在跨 LLM 和跨任务场景下的迁移表现。不迁移设置（来源与测试匹配） 取得了最大增益，主导热力图的对角线。跨 LLM 迁移（相同任务、不同骨干）紧随其后——在相同任务上分析轨迹 产生的元技能相似，不受底层 LLM 影响。跨任务迁移（相同 LLM、不同基准）同样展现出有竞争力的性能， 验证了进化过程明确提炼出的是任务无关的编排策略，而非领域专用技巧。

尤为关键的是，进化后的元技能并非仅仅是任务特定的修补：它们蕴含了可泛化的编排策略。 只有同时跨 LLM 和任务分布进行迁移（面板 C）时，增益才相对较弱，这反映了完全分布外泛化的极端难度—— 但即便如此，迁移后的技能仍维持在 Skill-MAS 初始版基线之上。

元技能进化：一条连贯的学习弧线

在此轨迹中，技能进化描绘了一条连贯的弧线：从证据如何组织框架，到如何裁决，再到如何补救失败。 模块 1 首先建立认知脚手架——约束优先级排序与扇出检索将分解从平面的子任务划分，转化为富含结构线索的关系搜索规划。 在此基础上，模块 2 以校准评估取代脆弱的二元检查——即带部分证据兜底的加权约束满足。 最后，模块 3 提升编排能力——跨实体桥接与合并节点重执行，将集成阶段从被动聚合转变为主动证据恢复。 综合来看，这些更新勾勒出一条渐进式的元技能进化路径：从分解设计，到智能体级认知控制，再到系统级韧性。

关键发现

Skill-MAS 通过进化的元技能一次性生成 MAS，在自动化 MAS 范式中实现了最佳成本-性能权衡。即便 Skill-MAS 初始版已展现出与强基线相当的竞争力；迭代进化在所有四个元智能体骨干上带来了最大的增益。

核心洞见

元技能提炼出策略级的编排原则（而非任务特定的修补），使得元技能能够在未见基准和不同 LLM 之间稳健迁移。用持续进化的架构知识增强静态元智能体，是打破能力-经验两难困境的关键。

综合总结

《多智能体优势的幻象》对六个代表性自动化 MAS 展开了系统性评估，涵盖多个 LLM 和基准。 结论令人警醒：CoT-SC 通常以极低的成本匹敌或超越这些框架。架构解构进一步揭示了症结所在—— 功能崩溃为集成、角色冗余、从不学习的启发式搜索，以及对任务结构视而不见的路由策略。

Unified-MAS 解决了 MAS 设计的 "做什么"问题。它将领域专用节点合成与拓扑编排解耦，离线检索开放世界知识 生成专家级节点（例如，法律要件提取器而非 "Expert1"）， 这些节点可直接插入现有自动化 MAS，无需修改其编排器，以更低的成本带来一致的性能提升。

Skill-MAS 解决了 "如何做"问题。它将编排抽象为可进化的元技能，通过多轨迹推演和选择性反思加以优化， 让冻结的前沿元智能体无需参数更新即可跨任务积累编排经验， 以中等推理成本实现强劲性能，且技能可在 LLM 和基准之间迁移。

综合来看，最核心的教训在于：多智能体协作本身并非固有缺陷，当今的自动化 MAS 失败的原因清晰可辨—— 它们在缺乏学习的情况下盲目搜索，在毫无根基的情况下生成节点，最终收敛于表面的复杂。 外部检索将节点生成从幻觉命名转化为领域工作流设计；可进化的元技能将编排搜索从重复的盲目迭代转化为经验驱动的优化。 不回避问题的诊断、领域深度和进化广度，三者各自不可或缺——唯有三者结合，自动化 MAS 才能开始兑现其迄今为止仅仅停留在纸面上的承诺。

未来方向：我们该走向何方？

基于上述三项研究，我们就自动化 MAS 未来的设计方向分享几点初步思考。

1. MAS 的机制可解释性

我们的架构解构表明，大多数 MAS 的所谓"协调"不过是装饰，并未真正奏效。我们需要工具来检验每个智能体角色是否对最终决策产生了可测量的因果影响。 这意味着从基于准确率的评估，转向结构保真度指标——以判断任务分解、并行化和专业化是否真正在生成的工作流中落地。

2. 超越文本：多模态 MAS

当前所有自动化 MAS 框架仅在文本模态中运行。然而，许多现实世界任务要求智能体能够同时推理图像、代码、结构化数据和工具输出。 将 Unified-MAS 节点合成范式扩展到多模态节点，并进化能够编排异构智能体池的元技能， 代表了一个自然而富有影响力的研究方向。

3. 从集中式到分布式进化

我们当前的工作集中于集中式编排，即单一元智能体控制 MAS 拓扑。然而，分布式智能体团队系统（如 Claude Code Agent Teams） 提供了替代的扩展特性。一个关键开放问题是元技能范式是否可以扩展到协同进化分布式协调协议， 使得智能体能够在没有中央控制器的情况下学习协商、委托和协作。
