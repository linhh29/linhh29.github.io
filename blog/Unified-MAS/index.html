Abstract

Automatic Multi-Agent Systems (MAS) generation has emerged as a promising paradigm for solving complex reasoning tasks. However, existing frameworks are fundamentally bottlenecked when applied to knowledge-intensive domains (e.g., healthcare and law). They either rely on a static library of general nodes like Chain-of-Thought, which lack specialized expertise, or attempt to generate nodes on the fly. In the latter case, the orchestrator is bound by its internal knowledge limits and must simultaneously generate domain-specific logic and optimize high-level topology — a severe architectural coupling that degrades overall system efficacy.

We propose Unified-MAS, which decouples granular node implementation from topological orchestration via offline node synthesis. Unified-MAS operates in two stages: (1) Search-Based Node Generation retrieves external open-world knowledge to synthesize specialized node blueprints; and (2) Reward-Based Node Optimization uses a perplexity-guided reward to iteratively enhance bottleneck nodes. Extensive experiments across four specialized domains show that integrating Unified-MAS into four Automatic-MAS baselines yields a much better performance–cost trade-off, achieving up to a 14.2% gain while significantly reducing costs. Further analysis reveals robustness across different designer LLMs and generalizability to mathematical reasoning.

Key Contributions

A New Decoupling Paradigm

We identify why Automatic-MAS fail in specialized domains and propose decoupling what each agent does (node implementation) from how agents connect (topology orchestration).

Search-Based Node Generation

Multi-dimensional keyword extraction and strategy-driven web search (Google, Scholar, GitHub) ground node synthesis in real domain knowledge — overcoming LLM parametric limits and hallucinated sub-agents.

Perplexity-Guided Optimization

A reward signal combining improvement and consistency scores identifies bottleneck nodes and iteratively refines their internal logic until the pipeline converges to reliable expert-level behavior.

Universal Plug-in for Automatic-MAS

Generated nodes plug into MAS-Zero, AFlow, ScoreFlow, and MAS² without changing their orchestrators — elevating general Automatic-MAS into domain experts with consistent gains across four LLM backbones.

Better Performance at Lower Cost

Domain-specific nodes replace inefficient generic reasoning chains, enabling fewer but more effective steps. Unified-MAS-enhanced systems sit on a superior performance–cost Pareto frontier across all evaluated settings.

Why Existing Automatic-MAS Fall Short

Static General Nodes

CoT, Debate, Self-Refine — powerful on general benchmarks, but the orchestrator merely stacks generic patterns without capturing legal, clinical, or financial nuance.

Dynamic On-the-Fly Nodes

MetaAgent, EvoAgent, AOrchestra invent sub-agents at runtime — but produce superficial nodes like "Expert1" without grounding, and distract the orchestrator from topology search.

Unified-MAS (Ours)

Offline synthesis of expert nodes (e.g., Legal_Element_Extractor, Liability_Reasoning) via external retrieval, then plug into any orchestrator for topology-only search.

Unified-MAS Approach

Unified-MAS acts as an offline node synthesizer before any Automatic-MAS topological search. It expands the node library from a static Vfix to a domain-adaptive Vdomain, then hands the enriched library to existing orchestrators unchanged.

Stage 1 · Keyword Extraction

Sample validation examples and extract keywords across 7 dimensions: Domain, Task, Entities, Actions, Constraints, Desired Outcomes, and Implicit Knowledge.

Stage 1 · Strategy Search

Four search strategies target background knowledge (Scholar), system architecture (Google), code repos (GitHub), and evaluation benchmarks — then aggregate into node blueprints.

Stage 2 · Reward Scoring

Execute nodes on validation data; compute perplexity-guided rewards measuring each node's improvement and consistency contribution to the final answer.

Stage 2 · Bottleneck Refinement

Identify the lowest-reward node each epoch, refine its prompts or add sub-calls, and repeat until performance converges — typically within 10 epochs.

Specialized Benchmarks

We evaluate on four highly specialized domains where manual MAS consistently outperform generic Automatic-MAS:

TravelPlanner
Constrained multi-day travel planning with commonsense & hard constraints
HealthBench
Healthcare diagnosis evaluated by LLM-as-a-Judge rubric scoring
J1Bench
Legal judgment — synthesizing conflicting testimonies into verdicts
DeepFund
Financial stock-market decision-making with accuracy evaluation

Baselines

Automatic-MAS (pre-defined nodes): MAS-Zero, AFlow, ScoreFlow, MAS²
Automatic-MAS (dynamic nodes): MetaAgent, EvoAgent, AOrchestra
Manual domain-specific MAS: PMC, Diagnosis-MAS, Court-MAS, DeepFund-MAS

Orchestrator LLMs: Gemini-3-Flash, GPT-5-Mini, Qwen3-Next-80B, DeepSeek-V3.2.

Main Results

The Domain Barrier

As shown in Table 1, manual domain-specific MAS consistently outperform all Automatic-MAS baselines across four orchestrator LLMs. With Gemini-3-Flash, Manual MAS averages 40.99% vs. 28.69% for Vanilla and 31.39% for the best dynamic method (AOrchestra) — confirming that domain expertise is critical in knowledge-intensive tasks. Dynamic methods like MetaAgent and EvoAgent show flashes of potential on J1Bench but fail catastrophically on TravelPlanner, often performing worse than the Vanilla baseline.

Analysis & Insights

(a) What Good Nodes Look Like

On J1Bench, dynamic methods like EvoAgent generate lazy ensemble nodes ("Expert1", "Expert2") without domain grounding. Unified-MAS produces a structured judicial pipeline:

Legal_Element_Extractor Fact_Analyzer Liability_Reasoning Damages_Calculator Judgment_Drafter

Observation

Dynamic node generation often resorts to superficial prompt-level voting. Unified-MAS nodes encode traceable, legally grounded reasoning stages grounded in retrieved statutes and case analysis patterns.

Insight

External retrieval is the key differentiator — it transforms node generation from hallucinated naming into expert workflow design.

(b) Robustness Across Designer LLMs

Unified-MAS elevates baselines regardless of which LLM serves as the "Designer" for node synthesis:

Observation

Gemini models synthesize concise macro-level workflows (~5–6 nodes). GPT-5-Mini prefers micro-level granularity (~10 nodes). Despite these topological preferences, Unified-MAS consistently drives substantial gains with all Designers.

Insight

Unified-MAS is not bottlenecked by any single LLM — it is a robust, model-agnostic node synthesis pipeline.

(c) Generalizability to Math (AIME24 & 25)

Even in general mathematical reasoning, Unified-MAS improves all baselines. With GPT-5-Mini + AFlow, accuracy rises from 59.18% to 67.35% (+8.17%); with DeepSeek-V3.2 + MAS², gains reach +15.65% (51.02% → 66.67%) while cost stays comparable.

Insight

While gains are more modest than in knowledge-intensive domains, Unified-MAS successfully synthesizes reasonable mathematical reasoning nodes — demonstrating broad applicability beyond specialized domains.
